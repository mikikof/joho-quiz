# 結果回収＆ランキングのセットアップ（10〜15分）

生徒の受験結果を Google スプレッドシートに集め、先生用ページでランキング表示するための準備です。
一度だけ設定すれば、あとは生徒がリンクから受験するだけで自動でたまっていきます。

## 仕組み

```
生徒のブラウザ(GitHub Pages) ──POST──▶ Apps Script ──▶ 先生のスプレッドシート
先生の teacher.html ◀──JSONP（ランキング）── Apps Script ◀── 同じシートを集計
```

## 手順

### 1. スプレッドシートを作る
1. [Google スプレッドシート](https://sheets.google.com) で新規作成（名前は「情報スピードクイズ 結果」など）。

### 2. Apps Script を貼り付ける
1. スプレッドシートのメニュー **拡張機能 → Apps Script** を開く。
2. 最初のコード（`function myFunction() {}`）を全部消す。
3. このフォルダの **`Code.gs` の中身を全部コピーして貼り付け**、保存（💾）。

### 3. ウェブアプリとしてデプロイ
1. 右上の **デプロイ → 新しいデプロイ**。
2. 歯車⚙ → **ウェブアプリ** を選択。
3. 設定：
   - 説明：任意（「情報スピードクイズ受信」など）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. **デプロイ** → 初回は権限の確認が出るので承認する。
   （「このアプリは確認されていません」→ 詳細 → （安全でないページ）に移動 → 許可）
5. 表示される **ウェブアプリ URL**（`https://script.google.com/macros/s/…/exec`）をコピー。

### 4. config.js に貼る
このフォルダの `config.js` を開き、`endpoint` にコピーした URL を貼る：

```js
window.APP_CONFIG = {
  endpoint: "https://script.google.com/macros/s/AKfy…/exec"
};
```

### 5. 動作確認
- ブラウザでその URL をそのまま開いて「情報スピードクイズ 受信サーバーは稼働中です。」と出れば OK。
- `index.html` を `?a=1` 付き（teacher.html の「配布」で作るリンク）で開いて1回受験 → スプレッドシートに行が増えるか確認。
- teacher.html の「②ランキングを見る」→「集計する」で順位が出れば完成。

## よくあるつまずき

| 症状 | 対処 |
|------|------|
| 結果が届かない | デプロイの「アクセスできるユーザー」が**全員**か確認。URL は末尾 `/exec`（`/dev` ではない）。 |
| ランキングが「取得できません」 | 同上＋`config.js` の endpoint が正しいか。コードを直したら**新しいデプロイ**（編集→デプロイを管理→バージョン新規）を忘れずに。 |
| 文字化け | スプレッドシートはそのままで問題なし。CSV 書き出し時は UTF-8 を選ぶ。 |

## クイズごとにタブが自動で分かれます（集計の分離）

このスクリプトは、**課題名ごとに専用タブ（シート）を自動作成**して結果を振り分けます。

- 配布時に付けた「課題名」が、そのままタブ名になります（例：`6/15 第1回 標本化クイズ`）。
- 違う課題名で配れば**別タブ**に、同じ課題名なら**同じタブ**に集約されます。
- `teacher.html` の「② ランキング」では、タブを選ぶだけでそのクイズの順位が出ます。
  （タブ一覧は自動取得。新しいクイズを足したら「一覧更新」を押す）
- タブ名に使えない記号（`: \ / ? * [ ]`）は自動で `-` に置き換えます。

> 短い単発クイズ（2〜5問）を何回も配って、回ごとにタブで分けて集計する使い方に向いています。

## 記録される列（各タブ共通）

受信日時／課題名／クイズID／氏名／学籍番号／スコア／正解／問題数／正答率(%)／合計タイム(秒)／
離脱回数／離脱(秒)／再提出／開始時刻／提出時刻／UA

> **離脱回数・離脱(秒)** … 受験中に別タブ・別アプリへ移った回数と時間。カンニングの目安として記録しているだけで、自動で減点はしません。
